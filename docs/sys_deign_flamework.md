# Klynx 系统设计框架

## 1. 项目简介
**Klynx** 是一个高级、高度可定制的自主智能体（Autonomous Agent）框架。它旨在与全球顶级的 LLM 服务提供商（如 OpenAI, Anthropic, Google, DeepSeek 等）无缝集成，并提供强大的工具调用基础设施和内置的交互式终端界面。

## 2. 核心技术栈
- **核心编排**: 基于 [LangGraph](https://langchain-ai.github.io/langgraph/) 构建核心状态机，实现代理的递归推理、记忆、错误反思以及持久化上下文循环。
- **大模型适配**: 基于 [LiteLLM](https://github.com/BerriAI/litellm) 实现统一的 LLM 路由，支持“编写一次代码，使用任何模型”。
- **UI & 交互**: 包含内置的富文本终端用户界面（TUI），用于流式控制台交互。

## 3. 主要特性与功能模块
- **通用 LLM 路由**: 代理可以无缝切换不同的底层模型引擎。
- **高级代理架构**: 支持复杂的任务规划、执行及自我纠错机制。
- **强大的工具环境**: 原生集成了浏览器自动化（Playwright）、操作系统终端交互、本地文件系统查询以及网络搜索功能。
- **网络代理抗性**: 内部集成了处理本地网络环境的功能，优雅处理 SSL 等拦截问题。

## 4. 目录结构预期
- `agent/`: 智能体核心逻辑及状态机实现
- `model/`: 模型配置及适配逻辑
- `kbase/`: 知识库及存储处理
- `tools/` (或内建于 agent): 工具集合实现
- `tui_app.py` / `main.py`: 启动入口与终端 UI 实现
